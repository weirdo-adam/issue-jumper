# Security Policy

## Supported Versions

Only the latest published release is supported for security fixes.

## Reporting a Vulnerability

Please report security issues privately instead of opening a public issue.

Use GitHub's private vulnerability reporting if it is available for this repository. If it is not available, contact the maintainer through the repository owner's GitHub profile.

Include:

- A clear description of the issue.
- Steps to reproduce or a minimal proof of concept.
- The affected version, platform, and command used.
- Any known workaround.

Do not include credentials, tokens, or private repository information in reports.
